Detailed Description

Defines command queue, an element is free if op_code field is not invalid.

Defines commands enqueued for flash access. At any point of time, this queue has one or more flash access operation pending if the count field is not zero. When the queue is not empty, the rp (read pointer) field points to the flash access command in progress or to requested next. The queue implements a simple first in first out algorithm. Data addresses are assumed to be resident.
