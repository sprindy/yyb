本文档主要引导您从我们提供的nrf51822demo开始，配合离线测试工具Firefly，一步一步的绑定成功，然后再使用手Q做最后测试的过程。
虽然Firefly绑定成功过，不代表此设备可以投入生产。但还是强烈建议先用Firefly测试，因为只要做少量修改，就能适最终产品。

Step 1. 编写设备代码

1.1 配置蓝牙广播

QQ物联的蓝牙广播标准如下

名称 值 作用
ServiceUUID 0xFEBA 标识QQ物联
Write(0x08) Characteristics UUID 0000fec7-feba-f1f1-99c0-7e0ce07d0c03 App向Device传输数据
Indicate(0x20) Characteristics UUID 0000fec8-feba-f1f1-99c0-7e0ce07d0c03 Device向App传输数据
Read(0x02) Characteristics UUID 0000fec9-feba-f1f1-99c0-7e0ce07d0c03 Device不发广播时标识设备
manufature specific data 以 PID(4字节)开头，以MAC(6字节)结尾 Device发广播时标识设备

ServiceUUID和Characteristic的值是固定的，按照格式填写即可。manufature的计算如下：比如某设备PID=1000000352(0x3B9ACB60)，MAC地址=F2:AF:0C:C1:FC:FD，则manufature字段是{0x60, 0xcb, 0x9a, 0x3b, 0xfd, 0xfc, 0xc1, 0x0c, 0xaf, 0xf2}。

设置完后，打开 firefly，点击扫描。如配置正常，就可以看到设备出现在搜索结果中；如果没有找到，请用LigthBlue检测上述配置是否正确。

对于您的开发板，只需要修改FT_LICENSE。FT_LICENSE的生成非常简单，在 firefly发现设备后，回复“+数字”得到该设备的license。

代码中定义了一个结构体，用来储存上面的值

typedef struct _QQ_SEC_INFO {
    uint8_t  license[FT_MAX_LICENSE_LEN];
    uint16_t licenselen;
    uint8_t  din[DIN_LEN];
    uint32_t pid;
    uint8_t  authkey[AUTH_KEY_LEN];
    uint8_t  pukstr[PUK_STR_BUFFER_LEN];
}QQSecInfo;

我们把所有数据储存在此结构体中，以便实现load/save操作。同时，也方便在批量生产时，为每一台设备写入信息。

1.3 实现fd_engine.h中的函数

fd_engine.h中有部分函数是交给硬件厂商实现的，这部分函数都是以fh_xxx开头。

  • int8_t fh_send(uint8_t* datum,uint16_t length); 这个函数是SDK在发送数据时调用的。厂商将数据发送完毕后，需要调用void fd_flush(struct fd_engine_s* engine);告知SDK，否则SDK不会继续发送数据。
  • void fh_load_mac(uint8_t* zone);返回设备的mac地址。比如我们这台设备返回的是”F2AF0CC1FCFD”，是一个12字节、没有\0结尾的字符串。注意顺序和大小写
  • fd_save_din/fd_load_din需要用到NVRAM或Flash
  • license、authkey、puk等参照demo实现，用我们上面定义的结构体即可
  • 其它方法根据需要自行实现或留空

1.4 调用fd_engine.h中的函数

实现了fd_engine.h中的函数后，工程就可以编译通过。但是要让SDK正常运行，还需要按顺序调用下面方法。

  • 分配一块空间，调用struct fd_engine_s* fd_init(uint8_t* zone,uint16_t size);创建一个全局fd_engine_s对象
  • 设备连接/断开时，调用fd_connect/fd_disconnet
  • 在接收数据的地方，调用int8_t fd_received(struct fd_engine_s* engine, uint8_t* datum,uint16_t length);，将收到的数据传给SDK

2.3 添加功能

我们现在做一个简单的亮灯功能：通过手机让开发板上面的灯亮起来。

在开发板实现int8_t fh_set_property(ff_property_t* prop);方法

int8_t fh_set_property(ff_property_t* prop)
{
    leds_ctrl(mLED_3, 2); //亮灯
    return 0;
}

App和开发板的数据交换，都是通过ff_property_t结构体。
当SDK收到手机发过来的请求，会回调上面这个函数。参数prop中比较重要的是prop_id，相当于方法名；value可以理解为参数。
上面代码非常简单，只要收到任何消息，就亮灯表示已收到。

点击send。如果一切正常，现在应该可以看到开发板上灯有闪动。

2.5 手机接收数据

现在我们实现另一个功能：按下开发板按钮，手机上收到一条消息。代码如下

static void button_event_handler(uint8_t pin_no, uint8_t button_action)
{
if (pin_no == mBUTTON_3 && button_action == APP_BUTTON_RELEASE)
  {
        char *value = "btn 3 press";
        ff_property_t ffprop;
        memset(&ffprop,0,sizeof(ffprop));
        ffprop.seq_no = 1;
        ffprop.len = strlen(value) + 1;
        ffprop.prop_id = 100000085;
        ffprop.value = (uint8_t*)value;

        fd_set_property(g_fd_engine, &ffprop);
    }
}

button_event_handler是开发板上按键回调函数，fd_set_property是设备向手机发送数据到方法。
按下mBUTTON_3，看到这样的消息提示，证明接收成功

至此，设备已基本可用。接下来，您只需要申请我们内测，并修改少量工程里面的数据，就可以自行接入到QQ物联平台。

3.3 替换demo中的配置

您需要将demo中的配置替换为您产品使用的值。需要替换的有FT_DEFAULT_PID，FT_DEFAULT_AUTH_KEY，FT_DEFAULT_PUK，FT_LICENSE。

FT_DEFAULT_PID是您申请的PID。

下载服务器公钥，打开文件，将内容拷贝到key_tools中，生成authkey和puk。

替换FT_DEFAULT_AUTH_KEY，FT_DEFAULT_PUK。

key_tools证书生成批量licnece，替换FT_LICENSE。

4. FAQ

Q：提示“设备未认证”？

A：你设备的mac地址没有在平台上登记，或登记后还没有审核通过。每次添加设备都需要重新审核

Q：提示“绑定失败，蓝牙设备注册失败，错误码：46”

A：请以下步骤确认：1、firefly中的s_public.pem、public.pem和网站上的一致；2、firefly可以绑定成功。
如果你是用的demo里面的PID，请不要重新生成产品公钥。如果已生成，请先卸载firefly，重启设备（排除document下有残留文件），然后重装，并按照上面的步骤重新生成license。

Q：post_result(ret_code=33)

A：FireFly目前不支持其他公私钥，请使用我们demo你们的authkey、puk、pid，并把mac地址发给我们代为登记。官网上的key_tools能帮助生产其他pid相应的authkey、puk、licence。

Q：第一次绑定成功，测试一切正常，下次却连接不上

A：请确认fd_load_din/fd_save_din已正确实现，且din的值保存到NVRAM或Flash中。绑定成功后SDK会调用fd_save_din，每次连接时会调用fd_load_din。如果不是绑定时保存的din，就会出现连接不上的情况

Q：我绑定的时候提示“该设备已被QQ xxx绑定”

A：让对方解绑

Q: 手Q绑定时提示“请检查设备是否正常，确保设备在手机附近”

A：可能是mac地址没有登记，或者二维码中sn对应了其它设备的mac地址。请在平台上下载登记表检查。

Q：代码里FT_SET_DEFAULT是做什么的

A：第一次运行时我们会打开这个宏，从代码里面读取puk等信息并写人到flash；下一次运行我们会关掉这个宏，这样就直接从flash里面获取。量产时一般由别的工具写入flash，所以应予以关闭

Q：seq_no和prop_id有什么规定吗？

A：seq_no一般从1开始递增，没有特殊规定，视应用而定；prop_id不能随便使用，以后台分配为准。

Q：蓝牙登记列表格式是怎样的

A：CSV格式，第一列sn，第二列mac地址

0280E10034120000,02:80:E1:00:34:12
0480E10034120000,04:80:E1:00:34:12
0402EC2500FD0000,04:02:EC:25:00:FD
20736A1010010000,20:73:6A:10:10:01

sn要求16位，可以是任意的由数字和字母组成。示例中用的MAC地址，真实环境不建议这样，以免被人伪造sn。

Q：广播字段可以包含其他数据吗

A：QQ定义广播的manufature字段前4字节为PID，后6字节为MAC。可以在中间放其他数据

Q：提示”绑定失败，请确认设备电源已打开。1002”

A：广播设置错误，或PUK、AUTH_KEY等长度不对。请检测代码。

Q：支持绑定的手Q版本

A：Android ≥ 5.6，iOS ≥ 5.7

Q：fh_get_property如才能调用

如需让fh_get_property被调用，H5发送代码，apiName需要将set_data_point改为get_data_point，此接口不会把value传到设备中。

Q：各种错误码的含义

A：详见网站 http://iot.open.qq.com/wiki/#!SDK/Errcode.md
